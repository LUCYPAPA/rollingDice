# 好婆叫侬来白相 · 产品 PRD v1.0

**产品类型**：微信小程序 · 民间家庭互动游戏集（单机 + 联机）
**核心受众**：家庭聚会场景，2人以上，老少皆宜
**设计原则**：轻、快、不用解释——拿起手机就能玩

---

## 一、产品概述
收录民间互动游戏，搬到线上，四世共乐。
第一版：源自外公外婆家的祖传骰子游戏。六颗骰子，一只大碗，红4是关键。线下传承几十年，2026年搬到手机上。联机版让不在同一桌的家人也能一起玩。

---

## 二、核心游戏规则

**基础道具**：6颗骰子 · 1只虚拟碗 · 筹码积分

**底池机制**：每位玩家投入底注（默认32点）→ 轮流摇骰子 → 触发牌型从底池拿点 → 底池清零本轮结束

### 两种游戏模式

**经典版（阿爹好婆原版）**
- 每轮底池清零后，所有玩家重新各投入底注开始新一轮
- 无淘汰机制，随时加入或退出
- 适合家庭聚会轻松玩

**对决版（改良版）**
- 每轮底注以当前存活玩家中余额最少者为基数，所有人等额上交
- 余额归零者出局
- 最后剩下唯一存活玩家获胜
- 适合想分出胜负的场景

## 三、用户旅程

### 旅程 A · 首次使用（联机）

```
打开小程序
    ↓
看到游戏主界面（Canvas 骰子动画）
    ↓
点击「联机」
    ↓
弹出昵称/头像设置页
    ├─ 可用微信头像一键填入
    └─ 可自定义昵称（最多12字，经内容安全检测）
    ↓
确认 → 云端登录（获取 openid，昵称/头像安全检测）
    ↓
首次赠送100点
    ↓
【欢迎卡】停留1.5秒，展示产品概述：
    "收录民间互动游戏，搬到线上，四世共乐。
     首次联机赠送 100 点，祝开门红！"
    ↓
进入联机大厅（老玩家直接进大厅，不弹欢迎卡）
```

**说明**：openid 作为系统唯一用户标识，玩家不可见，仅用于余额账本与历史记录关联。

---

### 旅程 B · 创建房间（房主）

```
联机大厅
    ↓
点击「创建房间」
    ├─ 设置底注（默认32点）
    ├─ 设置最多人数（默认6人）
    └─ 可选：添加机器人（最多3个）
    ↓
生成5位房间号（当日有效）
    ↓
等待大厅：显示玩家列表 + 房间号
    ├─ 可分享房间号/链接给朋友
    └─ 至少2人后可点「开始游戏」
    ↓
开始游戏 → 从所有玩家账户扣底注 → 进入游戏
```

---

### 旅程 C · 加入房间（玩家）

```
联机大厅
    ↓
点击「加入房间」→ 输入5位房间号
或
点击朋友分享的链接（自动填入房间号）
    ↓
验证通过 → 进入等待大厅
    ↓
等待房主开始游戏
```

---

### 旅程 D · 游戏中

```
游戏进行中（轮流摇骰）
    ↓
当前玩家：「摇骰子」按钮可点
其他玩家：按钮灰色，等待观看
    ↓
点击「摇骰子」
    ├─ 云端生成6个随机骰子（防作弊）
    ├─ 广播 rolling 状态 → 所有人播放物理动画
    └─ 动画结束后广播结算结果
    ↓
显示牌型 + 喊法 + 奖励点数
    ↓
    ├─ 底池未清零 → 点「下一位」→ 换人继续
    └─ 底池清零 → 触发轮次结算弹窗（所有玩家同时弹出）
                    ↓
        弹窗内容：本轮底注、底池总额、各玩家余额变动
        ├─ 上一轮收尾玩家（拿光底池的下一位）：
        │   显示「开始新一轮」按钮
        └─ 其他玩家：
            显示「等待 XXX 启动新一轮...」
                    ↓
        收尾玩家点「开始新一轮」→ 弹窗关闭
                    ↓
        经典版：重新收底注，开始新一轮
        对决版：余额归零者出局，继续对决；
                仅剩1人时游戏结束
    ↓
机器人回合：房主客户端自动触发，云端生成骰子
```

---

### 旅程 E · 游戏结束与结算

「局」= 一个房间从开始到结束的完整生命周期。

```
触发条件：
    A. 对决版仅剩1人存活（正常结束，底池理论清零）
    B. 房主点击「终止游戏」（中途结束，底池有余额）
    ↓
云端结算：
    ├─ 底池余额平分给所有真人玩家（余数给房主）
    ├─ 每人账户余额 += 本局净赢（chips）
    ├─ 写 balance_ledger 流水
    └─ 写校验摘要并锁定对局记录
    ↓
局后账单弹窗（每位玩家各自看到自己的版本）：

    ┌──────────────────────────────────┐
    │  本局结算单                        │
    │  房间号：XXXXX  |  2026-03-25      │
    │                                    │
    │  赛前余额  →  赛后余额             │
    │  100点      →  124点   (+24点 ✅)  │
    │  ─────────────────────────────    │
    │  本局每把记录：                    │
    │  第1把  小明  🎲4 4 2 1 3 5  +0   │
    │  第2把  小红  🎲4 4 4 2 1 3  +8   │
    │  第3把  我    🎲1 2 3 4 5 6  +16  │
    │  ...                               │
    │  ─────────────────────────────    │
    │  本局玩家最终余额（由高到低）：    │
    │  🥇 小红  148点                    │
    │  🥈 我    124点                    │
    │  🥉 小明   82点                    │
    │                                    │
    │  [查看历史记录]  [返回大厅]        │
    └──────────────────────────────────┘
```

**每把记录数据来源**：`game_logs.events` 数组，已有 diceValues、resultLabel、payout 字段，可直接展示。

> 机器人不占有真实余额，不参与底池平分，账单中标注「机器人」。

---

### 旅程 F · 历史记录

每局结算后，结账单以该玩家 openid 为维度保存一份快照，作为历史记录的数据来源。

```
入口 A：联机大厅 → 「我的手气记录」
入口 B：局后账单弹窗 → 「查看历史记录」
    ↓
列表页（倒序）：本账号参与过的所有对局
    每条记录显示：
    ├─ 日期 + 房间号
    ├─ 余额变动（绿色盈利 / 红色亏损 / 灰色持平）
    ├─ 同局玩家头像列（最多6个）
    └─ 结束方式（正常结束 / 房主终止）
    ↓
点击某条记录 → 展开本局完整结账单（同局后账单弹窗内容）
```

---

### 旅程 G · 单机模式

```
主界面 → 不点联机，直接设置玩家
    ↓
设置玩家人数（2-6人）+ 各自昵称
设置底注 + 模式（经典/对决）
    ↓
游戏流程与联机版一致，骰子本地生成
    ↓
游戏结束不写账户余额，积分清零重置
```
